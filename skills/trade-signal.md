# 技术面交易信号 — 交易卡生成

对 $ARGUMENTS（股票代码或名称）做**中短期（波段，数日至数周）**技术面分析，最终产出一张**交易卡**。

> 这套 skill 与价值投资 skill 是两套不同的纪律。价值投资看 10 年，本 skill 看数日到数周；价值投资靠"双源数据交叉验证"求严谨，本 skill 靠"历史回测"求严谨。**不要把两者的结论混用。**

---

## 核心原则（最高优先级）

1. **指标由 Python 算，判断由你做**——一切 RSI/MACD/均线/布林/ATR 数值必须来自 `tools/tech_indicators.py`，**严禁心算或凭记忆估算指标**。你只负责解读数字、综合判断。
2. **无回测不入卡**——任何写进交易卡的方向逻辑，都要参考 `backtest` 输出的历史胜率/盈亏比。胜率/盈亏比差的信号要在卡里如实标注"该信号历史表现一般"。
3. **风控优先于收益**——先确定止损（在哪里证明自己错了），再谈目标。没有清晰止损位的交易 = 不交易。
4. **客观、呈现两面**——遵守项目 CLAUDE.md 投研原则：摆数据→推逻辑→出结论；每个方向判断必须附**反面证据**（哪些指标不支持）。
5. **诚实留白**——数据不足（次新股、停牌、流动性差）时如实说"数据不足，不出信号"，不要用推测填满交易卡。
6. **三大技术面陷阱自检**：① 过拟合（别因为"历史完美"就重仓）；② 未来函数（引擎已只用 t 时刻及之前数据）；③ 叙事谬误（禁止主观读图说"主力吸筹"，只认可量化、可回测的信号）。

## 执行流程

### 第一步：确定标的代码

把用户输入转成 Yahoo Finance 代码：
- 美股：直接用 ticker（`NVDA`、`AAPL`）
- 港股：`代码.HK`（腾讯 `0700.HK`、美团 `3690.HK`）
- A股：沪市 `代码.SS`（贵州茅台 `600519.SS`）、深市 `代码.SZ`（宁德时代 `300750.SZ`）

如不确定代码，先 WebSearch 确认，再继续。

### 第二步：运行计算引擎（必须用 Bash，禁止心算）

```bash
# 技术面快照 + 信号旗标 + ATR风控建议
python3 ~/ai-berkshire/tools/tech_indicators.py analyze {代码}

# 信号历史回测（该股自身5年历史的经典信号胜率/盈亏比）
python3 ~/ai-berkshire/tools/tech_indicators.py backtest {代码}
```

若引擎返回 `error`（数据不足/代码错误），停止并告知用户，不要编造指标。

### 第三步：四视角审视（内部思考，制造张力）

把引擎给出的信号旗标，分别用四个视角过一遍——**四个视角会打架，这正是要的**：

| 视角 | 看什么 | 典型分歧 |
|------|--------|---------|
| **趋势跟踪者** | 均线排列、价 vs MA20/MA200、MACD、创新高 | "突破就追多" |
| **均值回归者** | RSI 超买超卖、布林带触边、KDJ、背离 | "RSI 80 别追，等回踩" |
| **量价分析者** | 量比、放量突破/缩量回调、OBV | "无量突破是假突破" |
| **风控官** | ATR 波动、止损空间、盈亏比、距高/低点位置 | "波动太大，仓位减半" |

综合规则（参考，非死板）：
- **ADX 市况是总开关**（最先看）：`ADX≥25`（趋势成立/强趋势）→ 趋势类信号（均线/MACD/突破）可信，**顺势**操作；`ADX<20`（震荡无趋势）→ 趋势信号**降权**，改用支撑/阻力区间**高抛低吸**（均值回归），**不要追突破**（多半假突破）；20-25 之间 → 趋势萌芽，减半仓观察。
- 多个视角共振同向 → 信号强，可给标准/确信仓
- 趋势与均值回归冲突（如上涨趋势 + RSI 超买）→ 多半给"等回踩入场"而非追高
- 量价不配合（突破但缩量）→ 信号降级为"观望/小仓试探"
- **支撑/阻力优先看强度**：引擎给的区间带 ★ 强度和触碰次数，★★★★以上、碰≥3次、贴整数关口的位才是硬位；只碰1次的弱位别当回事
- 布林挤压（带宽低分位）= 变盘前兆，方向未定 → 不预判方向，等放量突破再跟
- 回测显示该类信号在本股历史胜率 < 40% 且盈亏比 < 1 → 即使旗标好看也要降级并标注

### 第四步：输出交易卡

按下方模板输出。**所有数字来自引擎，不要自己改算**；方向、信心、是否等回踩由你综合判断。

### 第五步：询问是否保存

交易卡默认只显示。问用户是否保存到 `reports/交易信号/{代码}-{YYYYMMDD}.md`（中短期信号时效短，一般不必入库，除非用户要追踪）。

## 交易卡模板

```
┌─────────────────────────────────────────┐
│  📊 交易卡 · {名称}({代码})              │
│  {交易所} · {币种} · 截至 {日期}          │
└─────────────────────────────────────────┘

现价：{price}    | 60日区间：{low_60d} ~ {high_60d}（距高点 {pct_from_high60}%）
ATR：{atr}（{atr_pct}% 日波动） | 量比：{vol_ratio}
市况：{regime}（ADX {adx}）→ {趋势市:顺势操作 / 震荡市:高抛低吸、趋势信号降权}

▎关键价位（引擎区间，★=强度，碰N次=触碰次数）：
   阻力 {阻力区range} {★强度}（碰N次{,≈整数关口}）→ …更远 {次阻力}
   支撑 {支撑区range} {★强度}（碰N次{,≈整数关口}）→ …更远 {次支撑}

▎方向：【做多 / 做空 / 观望】     信心：★★★☆☆（X/5）
▎仓位建议：{试探仓 / 标准仓 / 确信仓}（占总仓 X%）

▎入场区间：{entry_low} – {entry_high}
   └ 是否等回踩：{是，等回踩至 MA20/支撑区 XX} / {否，当前可入}
▎止损：{stop}（{stop_pct}%，= 入场 ∓ 2×ATR 与 最近强支撑/阻力区 取更近者）
▎目标：T1 {target1}（{rr1}）→ 减半 | T2 {target2}（{rr2}）→ 清仓
▎盈亏比：约 {R}:1

▎触发依据（🟢 支持本方向）：
   • {共振的信号旗标 1}
   • {共振的信号旗标 2}
   • ...

▎反面证据（🔴 不支持，必须列）：
   • {相反或缺失的信号}
   • ...

▎历史回测参考（该股自身）：
   • {最相关策略}：胜率 X% / 盈亏比 X / 样本 N 笔
   • {如胜率/盈亏比差，明确写"该信号历史表现一般，谨慎"}

▎失效条件（论点作废，无条件离场）：
   • {如：放量跌破 XX} / {如：MACD 死叉} / {如：3日内未启动}

────────────────────────────────────────
四视角速记：趋势{🟢/🔴/⚪} 均值回归{} 量价{} 风控{}
多空计票：🟢{bull} vs 🔴{bear}（净 {net}）
────────────────────────────────────────
⚠️ 技术面信号为概率游戏，非确定性预测。本卡不含手续费/滑点，
   不构成投资建议。严格按止损执行，单笔风险勿超总资金 1-2%。
```

## 信心评分标准（★1-5，不含半星）

| 星级 | 条件 |
|------|------|
| ★★★★★ | 四视角全共振 + 放量突破 + 回测该信号胜率高、盈亏比>2 |
| ★★★★ | 三视角共振，量价配合，回测表现良好 |
| ★★★ | 主趋势明确但有 1 个视角矛盾，或量能不足 |
| ★★ | 信号混杂、方向不清，仅小仓试探价值 |
| ★ | 多视角矛盾或回测表现差，倾向观望 |

---

## 注意事项

1. **观望也是一种输出**——多空旗标五五开、或回测全差时，明确给"观望"并说明等什么信号再进，不要硬凑方向。
2. **多周期意识**——引擎用日线。若 60 日趋势与 20 日趋势矛盾（如长期跌、短期反弹），在卡里点明，波段方向以中期趋势为锚。
3. **财报/事件窗口**——若标的临近财报或有重大事件，技术信号会被基本面盖过，须提示"事件窗口，技术信号失效风险高"（可配合 /news-pulse）。
4. **仓位纪律**——确信仓也不等于满仓；单笔最大亏损（入场到止损）控制在总资金 1-2%，据此反推手数，而非据"看好程度"拍仓位。
5. **不预测，只应对**——交易卡是"如果到 A 就做 B"的应对方案，不是"它一定会涨到 X"的预测。
